什么是链接

链接是将各种代码和数据片段收集并组合成为一个单一文件的过程，这个文件可被加载（复制）到内存并执行。

链接可执行于编译、加载、运行三种情况下。

C语言文件编译过程

main.c -> main.i -> main.s -> main.o

首先，C预处理器将源程序 main.c 翻译成一个 ASCII 码的中间文件 main.i，然后编译器将 main.i 翻译成一个 ASCII 汇编语言文件 main.s，然后，汇编器将 main.s 翻译成一个可重定位目标文件。最后，链接器将经过同样步骤生成的 .o 文件组合创建一个可执行目标文件。

目标文件

  • 可重定位目标文件
  • 可执行目标文件
  • 共享目标文件

编译器和汇编器生成可重定位目标文件（包括共享目标文件）。链接器生成可执行目标文件。

可重定位目标文件

  • .text: 已编译程序的机器代码
  • .rodata: 只读数据
  • .data: 已初始化的全局和静态C变量
  • .bss: 未初始化的全局和静态C变量，以及所有被初始化为0的全局或静态变量。在目标文件中这个节不占据实际的空间，仅仅是一个占位符
  • .symtab: 一个符号表，存放在程序中定义和引用的函数和全局变量的信息
  • .rel.text: 一个 .text 节中位置的列表，当链接器把这个目标文件和其它文件组合时，需要修改这些位置
  • .rel.data: 被模块引用或定义的所有全局变量的重定位信息
  • .debug: 一个调试符号表， -g 调用编译器才会得到
  • .line: 原始 C 源程序中的行号和 .text 节中机器指令之间的映射， -g 调用编译器才会得到
  • .strtab: 一个字符串表，内容包括 .symtab 和 .debug 节中的符号表，以及节头部中的节名字

.symtab 符号表包含模块 m 定义和引用的符号的信息，共有三种符号：

  • 由模块 m 定义并能被其它模块引用的全局符号。对应于非静态的 C 函数和全局变量
  • 由其它模块定义并被模块 m 引用的全局符号。对应于其他模块中定义的非静态的 C 函数和全局变量
  • 只被模块 m 定义和引用的局部符号。它们对应于带 static 属性的 C 函数和全局变量。这些符号在模块 m 中任何位置都可见，但是不能被其它模块所引用

注意，在这里， static 属性的函数和全局变量不能被其它模块访问，即这些数据是私有的。

如何解析多重定义的全局符号

函数和已初始化的全局变量是强符号，未初始化的全局变量是弱符号。规则如下：

  • 1、不允许有多个同名的强符号
  • 2、如果有一个强符号和多个弱符号同名，那么选择强符号
  • 3、如果有多个弱符号同名，那么从这些弱符号中任意选择一个

链接分类

静态链接

静态链接器以一组可重定位目标文件和命令行参数作为输入，生成一个完全链接的、可以加载和运行的可执行目标文件作为输出。

为了构造可执行文件，链接器需要完成两个任务：

  • 符号解析。目标文件定义和引用符号，每个符号都对应于一个函数、一个全局变量或一个静态变量（即 static 属性声明的变量）。符号解析的目的时将每个符号引用正好和一个符号定义关联起来。
  • 重定位。编译器和汇编器生成从地址0开始的代码和数据节。链接器通过把每个符号定义与一个内存位置关联起来，从而重定位这些节，然后修改所有对这些符号的引用，使得它们执行这个内存位置。

但是，如果多个进程都要使用一个动态库，如果每个进程在内存生成的可执行文件都包含这个库，那么就会导致内存的大量浪费，所以就引出了共享库。

有了共享库，Linux 链接器还支持库打桩(library interpositioning)，它允许我们截获对共享库函数的调用，取而代之执行自己的代码。同样的，它可以在编译、链接、运行时执行。
