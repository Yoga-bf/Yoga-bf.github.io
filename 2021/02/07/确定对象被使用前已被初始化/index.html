确定对象被使用前已被初始化

发表于 2021-02-07 分类于 C++ ， Effective C++ ， 第一章：让自己习惯C++
初始化很重要
  • 为内置型对象进行手工初始化，因为C++不保证初始化它们
  • 构造函数最好使用成员初值列，而不要在构造函数本体中使用赋值操作。初值列列出的成员变量，其排列词语应该和它们在 class 中的声明次序相同
  • 为免除“跨编译单元之初始化次序”问题，请以 local static 对象替换 non-local static 对象。

永远在使用对象之前先将它初始化

对于内置类型以外的任何其它东西，确保每一个构造函数都将对象的每一个成员初始化。

注意要区分赋值（assignment）和初始化（initialization）

在成员初值列中初始化成员变量时，一般效率更高。因为在赋值版本中，首先调用了 default 构造函数为 theName，theAddress设初值，然后对他们赋予新值，而在初始化版本中，直接进行了 copy 构造，效率更高。
