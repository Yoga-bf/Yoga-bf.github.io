绝不在构造和析构过程中调用 virtual 函数

发表于 2021-02-07 分类于 C++ ， Effective C++ ， 第二章：构造、析构、赋值运算
在 base class 构造期间，virtual 函数不是 virtual 函数
  • 在构造和析构期间不要调用 virtual 函数，因为这类调用从不下降至 derived class (比起当前执行构造函数和析构函数那层)。

上述代码中，base class 是 Transaction，它的构造函数中使用了 virtual 函数 logTransaction，在 derived class 中， 由于它要调用 base class 的构造函数，在 base class 的构造函数中， logTransaction 是 base class 的 logTransaction，而不是 derived class 的，所以可能会导致未知错误。

即 base class 构造期间 virtual 函数绝不会下降到 derived classes 阶层。

原因是因为在 derived class 构造期间，base class 的构造函数早于 derived class 的构造函数执行，在 base class 构造函数执行时 derived class 的成员变量尚未初始化，如果此期间调用的 virtual 函数下降到 derived classes 层，derived classes 层的函数几乎必然取用 local 成员变量，而这些变量未初始化，这将导致错误。

修改方法是将 logTransaction 函数设置为 non-virtual 函数，这个函数接收参数来实现不同的过程，然后 derived class 在构造函数期间，给 logTransaction 函数传递参数，实现构造。
但是要清楚，传递的参数需要初始化，即不能是 “初期未成熟至 BuyTransaction 对象内部尚未初始化的成员变量”。 所以需要使用 static 来修饰变量。
