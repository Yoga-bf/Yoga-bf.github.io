C++构造、析构、拷贝语意学

发表于 2021-05-11 更新于 2021-05-12 分类于 C++ ， class ， 语意学
试想一下，一个 class object 从产生到消失、它发生了什么？在构造、复制和析构的时候，编译器又做了什么使得程序能够正确执行？

当执行构造、拷贝、析构函数时，编译器会提供什么服务？

众所周知，有的 class 是继承而来、有的 class 带有 member class object、有个 class 有 virtual function 还有的 class 有 virtual base class，特别的，有一些 pure virtual function 存在，对于 class 的 member function 还有一些特定的修饰符，当这些作用于 class 时，又会发生什么？

一些小知识

pure virtual function

抽象类 - cppreference.com，一般来说 pure virtual function 是用来当作接口，即每个继承 abstract class 的 derived class 都需要定义这个函数的实现，当然，不定义也是可以的，但是不定义的话，此 derived class 同样是个 abstract class。

神奇的是，pure virtual function 同样定义它的实现。

member function const

如果一个 member function 后面带有 const，那么就是说这个 function 不能修改 class object 的值

如果一个 function 是 const，那么就不能将 pointer 和 reference 作用到它身上，即不能将成员函数指针作用到它。如果一个函数不修改 class object 的值，建议写成 const。

继承体系下的对象构造

在继承体系下，对象构造一般经过一下几个步骤：

  1. 对于 member initialization list 中的 members 初始化操作会被放到 constructor 的函数体内，且初始化顺序和 member 的声明顺序相对应。
  2. 如果有 member 没有出现在 member initialization list 中，但是这个 member 有 default constructor，那么这个 default constructor 将会被调用
  3. 如果有 virtual table pointer，必须要设定初值，指向特定的 virtual table。
  4. 如果上一次的 base class constructor 必须被调用，则按照声明顺序调用
    1. 如果 base class 被列于 member initialization list，那么任何显式指定的参数都应该传递过去
    2. 如果 base class 没有 列于 member initialization list，而它有 default constructor，那么就调用
    3. 如果 base class 是多重继承下的第二或后继 base class，则要修改 this 指针的指向。
  5. 在之前，所有 virtual base class constructor 都必须被调用，从左到右，从最深到最浅
    1. 如果 class 被列于 member initialization list，那么如果任何显式指定的参数，都应该传递过去，若没有在 list 中，那么 class 有 default class constructor 的应该要调用
    2. class 的每一个 virtual base class subobject 的偏移位置（offset）必须在执行器可被存取
    3. 如果 class object 是最底层（most-derived）的 class，其 constructor 可能被调用，某些用以支持这一行为的机制必须被放进来

事实上，我们只要在 base class constructor 调用操作完成之后再调整 vptr 就好了。在上面的带 virtual base class 的 constructor 那里，可以看出，derived class 是需要先调用 base class 的 constructor 的，如果在调用之前修改 vptr，那么在 base class 的 constructor 里面又会修改 vptr，而且这种方式也不适合继承体系，继承体系应该先构造好 base class 再构造 derived class，那么 vptr 的修改也就应该放在 base class 的 constructor 后面了。

所以顺序如下：

  1. 在 derived class constructor 中，所有 virtual base classes 及上一层 base class 的 constructor 先被调用。
  2. 初始化 vptr(s)，指向相关的 virtual table(s)
  3. 如果有 member initialization list 的话，将在 constructor 体内扩展开，但是必须在 vptr 设定之后，防止 member Initialization list 里面调用 virtual function
  4. 执行用户的代码

析构语意学

析构与构造对应，主要是为了将类对象的资源释放

那在析构的时候，应该要怎么做呢？一个朴素的想法，按照构造函数相反的顺序进行释放资源。事实上，如果一个 class 没有定义 destructor，那么只有它的 member 具有 destructor 或者 它的 base class 具有 destructor 的情况下，编译器才会给它合成一个 destructor。

尽管 Point class 具有一个 virtual function，但是却不需要 destructor 的。

顺序为：

  1. destructor 的函数本体首先被执行
  2. 如果 class 拥有 member class objects，而后者拥有 destructors，那么它们会以声明顺序相反顺序被调用
  3. 如果 object 内耗一个 vptr，现在被重新设定，以指向合适的 base class 的 virtual table
  4. 如果有任何直接的（上一层）nonvirtual base classes 拥有 destructor，它们会以其声明顺序的相反顺序被调用
  5. 如果有任何 virtual base classes 拥有 destructor，而目前讨论的这个 class 是最尾端 (most-derived) 的 class，那么它们会以其原来的构造顺序的相反顺序被调用。
